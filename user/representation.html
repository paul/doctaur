GitHub API

This describes the resources that make up the official GitHub API v3. If
you have any problems or requests please contact
support.

View the API Changelog for information on existing and
planned changes to the API.

Content Type

The GitHub API uses a vendor-specific mime type for web service requests and responses.

application/vnd.github.v3+json

We also accept the common application/json as an alias, but strongly discourage its use. It will always be mapped to the latest version of the API. If we upgrade the API at some point in the future, and you are relying on the previous version, bad things will probably happen.

User

A User represents a person or automated actor on GitHub. They may own and
contribute to repositories and belong to zero or more
organizations. Users have email addresses and ssh keys, as well
as other relevant profile data.

Representation

Attributes

login
  • string
The login name for the user
name
  • string
The user's meatspace name
last_login
  • iso8601
Date and time of the user's last login to github.com

Links

organizations_href
  • organization
[Organizations](#organization) of which this user is a public member
emails_href
  • email
The public [emails](#mail) registered to this user
ssh_keys
  • key
Public [SSH keys](#ssh_key) for this user

Retrieve current user

GET /user

Requires authentication

You can use /user to get the currently logged in user. This is a shortcut to /user/{login} for
your particular login.

curl -u "paul:{{password}}" \
     -H "Accept: application/vnd.github.v3+json" \
     https://api.github.com/user

Update current user

Requires authentication

To update your user profile data, you can PUT (or PATCH) some the user attributes you want modified.

curl -u "paul:{{password}}" \
     -H "Accept: application/vnd.github.v3+json" \
     -H "Content-Type: application/vnd.github.v3+json" \
     -d '{"bio": "The GitHub API Dude"}' \
     https://api.github.com/user

Fetch user by login

GET /users/{login}

To lookup a particular user's public profile data, you can find them by their login name.

curl -H "Accept: application/vnd.github.v3+json" \
     https://api.github.com/users/paul
